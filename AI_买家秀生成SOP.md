# AI 买家秀图片生成标准作业流程 (SOP)

**版本**：v1.1
**适用场景**：电商买家秀、对镜自拍、真实生活场景图
**目标**：从 AI 生成到可用素材，控制在 3 轮迭代以内，评分达到 B 级（50+）或 A 级（65+）

---

| 工具 | 路径 | 用途 |
|------|------|------|
| 蓝图生成 Skill | `~/.claude/skills/gpt-image-2-blueprint/` | 将需求转化为结构化 JSON 提示词 |

## 二、四步流水线

### Step 1：生成初版（蓝图 Skill）

**触发词**：`$image2-blueprint`

**输入**：向 Claude 描述你想要的场景
> 示例："生成一个男士睡衣的真实买家秀对镜自拍，要求慵懒随意，不能有 AI 感"

**输出**：Claude 会输出完整的 JSON 结构化提示词

**执行生成**：
```bash
# 保存提示词到文件
cat > prompt.txt << 'EOF'
[粘贴 Claude 输出的 JSON 提示词]
EOF

# 执行生成（替换为你的参考图路径）
python ~/generate_image.py prompt.txt "C:/path/to/reference.jpg"
```

**检查点**：
- [ ] 图片已成功生成（在 `~/.codex/generated_images/` 下能找到）
- [ ] 服装颜色、版型基本正确
- [ ] 没有明显的肢体畸形（如手指粘连）

---

### Step 2：客观扫描（Python 工具）

**命令**：
```bash
python ~/ai_audit_tool.py "生成图片的路径.png" --hands -o report.json
```

**输出示例**：
```json
{
  "exif": { "has_exif": false, "suspicious": true },
  "noise": { "score": 7, "assessment": "噪点水平正常" },
  "histogram": { "warm_bias": 23.2, "tone_assessment": "偏暖色调" },
  "hands": { "hands_detected": 1, "fingers_per_hand": [5], "alerts": ["正常"] },
  "risk_summary": { "risk_level": "MEDIUM", "risks": ["缺少 EXIF 元数据"] }
}
```

**解读**：
- `risk_level: HIGH` → 有硬性 AI 痕迹，必须重跑
- `risk_level: MEDIUM` → 可用，但需 LLM 主观复核
- `risk_level: LOW` → 客观指标通过，进入主观评分

---

### Step 3：主观评分（评分 Skill）

**触发词**：`$image-audit`

**输入**：上传图片 + 粘贴 Python 报告（如有）
> 示例："$image-audit 分析这张买家秀图的 AI 感，这是 Python 扫描报告：[粘贴 JSON]"

**输出**：8 维度评分表 + 综合等级

| 等级 | 分数 | 处理方式 |
|------|------|----------|
| A | 65-80 | 可用，轻微后期即可投产 |
| B | 50-64 | 可用，需针对性修复后重跑 |
| C | 35-49 | 有明显 AI 感，需重写提示词 |
| D | 0-34 | 不可用，重新设计场景 |

---

### Step 4：迭代修复

**根据评分报告，定位扣分项 → 收紧对应 JSON 字段 → 重跑**

常见修复对照表：

| 扣分项 | 修复策略 | JSON 中应加强的字段 |
|--------|---------|-------------------|
| 皮肤太光滑 | 增加极端面部细节 | `subject.face`: "粗大毛孔、青色胡茬、鼻翼油光、黑眼圈" |
| 手指粘连 | 精确描述每根手指 | `subject.pose`: "四指弯曲握住，指节骨清晰，拇指在右侧" |
| 面料无纹理 | 用比喻强化质感 | `outfit.top`: "像绉布/豆腐块表面的凸起颗粒" |
| 无阴影 | 强制光影指令 | `scene.shadow`: "必须在台面上投射清晰黑色投影" |
| 背景太干净 | 增加污渍杂物 | `scene.imperfections`: "牙膏渍、水垢、掉落头发" |
| 色调偏冷 | 锁死色温 | `rendering.color_grade`: "强制 2800K 暖黄，禁止冷色调" |

**重跑命令**：同 Step 1，使用修复后的提示词

## 五、常见问题 (FAQ)

**Q1：为什么不用 Midjourney/SD？**
A：GPT Image 2 对结构化 JSON 提示词的理解更强，且支持参考图一致性，更适合商品固定款式的批量生成。

**Q2：提示词太长会超 Token 吗？**
A：目前 5000 字以内 JSON 安全。若需更长，可分两段发：先传参考图 + 简短说明，再传完整 JSON。

**Q3：如何保持多张图的人物一致？**
A：Step 1 生成一张"人物锚点图"（白底正面照），后续每次生成时同时上传：锚点图 + 原参考图。

**Q4：生成的图没有 EXIF，会被平台识别为 AI 吗？**
A：部分电商平台会检测。可用 ExifTool 后期注入假 EXIF：
```bash
exiftool -Make="Apple" -Model="iPhone14,2" -DateTimeOriginal="2026:05:01 12:00:00" image.png
```

**Q5：Python 工具报 "codex 找不到"？**
A：修改 `generate_image.py` 中的 `codex_path` 为你的实际安装路径，通常在：
`C:/Users/<用户名>/AppData/Roaming/npm/codex.cmd`

**Q6：宣传图和买家秀有什么区别？模特表情应该怎么定？**
A：两者是完全不同的场景逻辑，不可混用同一套 prompt：

| 场景 | 目标 | 模特气质 | 皮肤细节 | 典型表情 |
|:---|:---|:---|:---|:---|
| **宣传图（淘宝主图）** | 吸引点击、促进转化 | 精神饱满、甜美/温和有亲和力 | 适度磨皮，气色健康 | 自然微笑、温柔恬静、可爱俏皮 |
| **买家秀（UGC）** | 营造真实感、降低决策疑虑 | 像真实普通人 | 极端真实（毛孔、胡茬、黑眼圈） | 疲惫、随意、无表演感 |

**关键原则**：宣传图模特不可有黑眼圈、眼袋、疲惫憔悴感；买家秀则必须有这些"瑕疵"才真实。

**Q7：为什么生成的图和参考图太像，像裁剪的？**
A：GPT Image 2 对参考图的 fidelity 很高。当 prompt 中出现"保持同款""保持一致"等措辞时，模型倾向于复制而非重新创作。解决方法是：
1. 参考图只用于约束**服装版型、颜色、面料**，不用于约束**模特、姿势、场景**
2. prompt 中明确写"模特面孔/姿势/场景必须完全重新设计，不可复制参考图"
3. 给 model 字段写入完全不同的面部特征描述（如：方圆脸、单眼皮、波浪卷发等）

**Q8：如何给模特"换脸"但保持服装不变？**
A：修改 prompt 中 `subject.model` 字段的面部特征描述，重新跑一轮即可。示例：
```json
"model": "年轻亚洲女性，面容必须与上一张完全不同：鹅蛋脸/方圆感，内双或单眼皮，略宽鼻翼，厚唇自然放松，波浪卷发低马尾，表情温柔恬静，必须是虚构合成人脸"
```
注意：`reference_image_note` 中应强调"保持服装款式但人物全新设计"。

**Q9：一个品类跑通了，如何扩展到其他品类？**
A：不同品类的视觉逻辑差异极大，需要重写 prompt 核心字段：

| 品类 | 风格定位 | 模特气质 | 背景策略 | 核心差异 |
|:---|:---|:---|:---|:---|
| 男士睡衣（成熟） | 简约品质感 | 成熟、松弛、无表演感 | 家居场景（沙发/床铺） | 慵懒、真实、低饱和 |
| 女士家居服（甜美） | 甜美少女感 | 活泼、甜美、有表演感 | 纯色背景+可爱道具 | 高饱和、精致、可爱 |
| 运动服 | 动感活力 | 阳光、健康、有肌肉感 | 健身房/户外 | 线条、张力、汗水感 |
| 商务正装 | 干练专业 | 精英感、自信 | 办公室/纯色 | 挺括、锐利、高级灰 |

不要试图用同一套 JSON 模板覆盖所有品类，应针对每个品类单独设计 `style`、`scene`、`model`、`rendering` 字段。

### 淘宝商品主图生成（7张主图 + 1张参数图）

**触发词**：`$image2-blueprint` → 选择 Mode E（淘宝商品主图）

**快速生成**：
```bash
# 基础6张（含模特首图+卖点底图+视频脚本）
python ~/taobao_image_kit.py \
  --name "商品名称" --desc "款式描述" --fabric "面料" --color "颜色" \
  --ref "C:/path/to/reference.jpg" --ratio 1:1 --slots 0-5

# 全部9张（含模特/视频/背面/透明底/参数图）
python ~/taobao_image_kit.py \
  --name "商品名称" --desc "款式描述" --fabric "面料" --color "颜色" \
  --ref "C:/path/to/reference.jpg" --ratio 1:1 --slots all
```

**Slot 说明**（基于猫人2万+销量爆款真实结构）：
| Slot | 内容 | 用途 |
|:--:|:---|:---|
| **0** | **模特场景首图** | 首图，AI无脸模特+沙发/床铺场景 |
| **1** | **卖点模特底图** | 模特站立全身，预留文字叠加区 |
| **2** | **主图视频脚本** | 15-30秒分镜脚本，3-4个关键帧 |
| 3 | 面料质感特写 | "rua一下"柔软手感、纤维纹理 |
| 4 | 做工细节 | 圆领/刺绣/腰头/裤脚/锁边 |
| 5 | **纯白底图** | 平台强制，RGB 255,255,255 |
| 6 | **背面/侧面** | 展示版型背面和裤型 |
| 7 | **透明底图** | 绿幕生成，需 remove_chroma_key 去底 |
| 8 | **参数图/尺码表** | 详情页首屏或主图第6张 |

**透明底图去底命令**：
```bash
python "~/.codex/skills/.system/imagegen/scripts/remove_chroma_key.py" \
  --input slot7_green.png --out slot7_transparent.png \
  --auto-key border --soft-matte --despill
```

### 淘宝详情页长图生成

**命令**：
```bash
python ~/taobao_detail_kit.py \
  --name "商品名称" --desc "款式描述" --fabric "面料" --color "颜色" \
  --ref "C:/path/to/reference.jpg" \
  --sections poster,emotion,size,data_viz,fabric,model,detail,scene,care,trust

# 覆盖面料技术指标（如非默认的猫人款参数）
python ~/taobao_detail_kit.py ... \
  --softness-value "46" --comparison-value "99" \
  --softness-level "5级一等面料" --antibacterial-level "7A级抑菌"
```

**板块说明**：
| 板块 | 内容 | 详情页位置 |
|:---|:---|:---|
| `poster` | 海报首屏+氛围文案 | 第1屏，视觉钩子 |
| `emotion` | **情感文案**（"像把云穿上身"） | 第2-3屏，情绪共鸣 |
| `size` | **尺码快选**（体重+身高对照） | 第3-4屏，降退货率关键 |
| `data_viz` | **面料数据可视化**（柔软度对比） | 第4-5屏，科技感信任 |
| `fabric` | 面料成分与工艺 | 第5-6屏，材质信任 |
| `model` | 模特场景多角度（3-4组姿势） | 第6-8屏 |
| `detail` | 细节卖点（圆领/刺绣/腰头/锁边） | 第8-9屏 |
| `scene` | 多场景穿搭 | 第9-10屏 |
| `care` | **洗护说明标签**（5个国标图标） | 第10-11屏，平台合规必备 |
| `trust` | 质检报告/售后保障 | 尾屏 |

### 小红书封面生成

**快速生成**：
```bash
python ~/xiaohongshu_cover_gen.py \

  --topic "笔记标题" \
  --product "产品名" \
  --color "颜色" \
  --ref "C:/path/to/reference.jpg" \
  --ratio 3:4 \

# 批量生成全部 4 种风格（供 A/B 测试选用）
python ~/xiaohongshu_cover_gen.py \

  --topic "笔记标题" \
  --product "产品名" \
  --ref "C:/path/to/reference.jpg" \
  --ratio 3:4 \
  --output cover.json
```

**风格选项**：
| 风格 | 适用场景 | 配色 | 主体 pose |
|------|---------|------|----------|
| `magazine` | 好物分享、生活方式 | 奶咖、米白、淡粉 | 产品自然摆放于床铺/沙发，搭配抱枕毯子 |
| `bold` | 干货清单、教程 | 亮黄、粉紫、薄荷绿 | 正面平铺或手持，纯色色块背景，主体突出 |
| `collage` | 合集、多品对比 | 牛皮纸黄、淡粉、灰蓝 | 多张小图拼贴，手账风背景，胶带贴纸点缀 |
| `cozy` | 深夜治愈、情绪 | 暖黄、深棕、焦糖 | 搭在沙发扶手或铺在床上，烛光/台灯，大面积暗部 |

**比例选项**：`1:1`（信息流方图）、`3:4`（竖版封面，默认）、`4:3`（横版）。

**关于文字**：GPT Image 2 中文渲染能力有限，建议：
1. 生成封面底图（无字）
2. 使用 Canva / 稿定设计 / PS 后期叠加文字

**质检标准**：
- 使用 `$image-audit` 的 **Xiaohongshu Cover Audit** 维度
- 视觉冲击力权重 1.5x，必须能在信息流中"停住手指"
- 预留排版空间，文字区域 ≤30%
